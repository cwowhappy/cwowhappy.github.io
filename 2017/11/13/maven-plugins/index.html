1. Maven常用插件列表

用户可以通过两种方式调用Maven插件goal：

  1. 将插件目标与Maven的生命周期阶段绑定，用户直接执行生命周期阶段就会执行相应的插件目标
  2. 直接指定要执行的插件目标（插件标识:插件目标）
Plugin 目标-Goal 功能
maven-compiler-plugin compiler:compile编译src/main/java目录下的源代码；compiler:testCompile编译src/test/java目录下的源代码 编译源代码
maven-resources-plugin resources:resources拷贝src/main/resources目录下的资源文件到output目录；resources:testResources拷贝src/test/resources目录下的资源文件到测试output目录；resources:copy-resources拷贝资源到output目录； 处理资源文件,过滤资源文件并替换其中的变量（如果变量在filter配置的属性文件中则替换，否则不做处理）；默认绑定到Maven的compile阶段
maven-jar-plugin Maven默认打包工具，默认绑定到Maven的package阶段
maven-surefire-plugin surefire:test执行项目的测试用例，默认绑定到Maven的test阶段 执行项目的测试用例
maven-archetype-plugin archetype:generate根据指定的archetype生成项目骨架 创建项目骨架
maven-shade-plugin 详情见单独的笔记 打包工具
maven-assembly-plugin 详情见单独的笔记 打包工具
maven-antrun-plugin run 在Maven项目中执行Ant任务
maven-dependency-plugin dependency:list列出项目的依赖列表；dependency:tree描绘项目依赖树；dependency:analyze检查分析项目依赖潜在的问题；dependency:copy-dependencies将项目依赖从Maven本地仓库复制到指定目录； 帮助用户分析项目依赖
maven-enforcer-plugin enforcer:enforce负责执行规则检查，默认绑定到生命周期的validate阶段；enforcer:display-info打印内置规则检测后的信息 强制参与项目开发的工程师遵守执行的跟Maven配置相关的规则（设定Java版本、设定Maven版本、禁止某些依赖等）
maven-help-plugin help:system答应所有可用的环境变量和Java系统属性；help:effective-pom打印项目的有效pom（合并了所有父pom的pom文件）；help:effective-setting打印项目的有效settings；help:describe打印任意Maven插件的信息 辅助工具
maven-release-plugin release:prepare准备版本发布，具体工作包括：检查是否有未提交的代码、检查是否有SNAPSHOT依赖、升级项目的SNAPSHOT版本至RELEASE版本、为项目打标签等；release:perform签出标签中的Release源码，构建并发布； 自动化项目版本发布，依赖于项目pom中的SCM信息
